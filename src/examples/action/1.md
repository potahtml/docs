---
title: Pipeline action for a form submit
description: Chain validation, fetch, and side-effects through one call site.
category: reactivity
api: action
import: pota
tags: [action, async, form]
---

`action(...fns)` returns a function whose stages run in order,
unwrapping promises along the way. Each stage receives the previous
stage's resolved value. The whole chain is owned by the surrounding
scope, so disposal cancels in-flight work, and rejections route to
the nearest `catchError`.

```jsx
import { action, render, signal } from 'pota'
import { Errored } from 'pota/components'

function Form() {
	const email = signal('')
	const status = signal('idle')

	const submit = action(
		() => email.read(),
		value => {
			if (!value.includes('@')) throw new Error('invalid email')
			return fetch('/subscribe', {
				method: 'POST',
				body: JSON.stringify({ email: value }),
			})
		},
		res => res.json(),
		result => status.write(`subscribed: ${result.id}`),
	)

	return (
		<form on:submit={e => (e.preventDefault(), submit())}>
			<input
				prop:value={email.read}
				on:input={e => email.write(e.currentTarget.value)}
			/>
			<button>subscribe</button>
			<p>{status.read}</p>
		</form>
	)
}

function App() {
	return (
		<Errored
			fallback={(err, reset) => (
				<div>
					<p>oops: {err.message}</p>
					<button on:click={reset}>try again</button>
				</div>
			)}
		>
			<Form />
		</Errored>
	)
}

render(<App />)
```
