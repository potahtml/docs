---
title: Animate by swapping CSS classes
description: Cross-fade between two animation classes and await the transition.
category: use
api: animate
import: pota/use/animate
tags: [animate, css, transitions]
---

`animateClassTo(element, oldClass, newClass)` removes the first
class, adds the second, and resolves once the resulting CSS
animation ends. If the new class doesn't trigger an animation, the
promise resolves immediately. Same idea for `animatePartTo` with
`::part()` shadows.

```jsx
import { animateClassTo } from 'pota/use/animate'
import { ref, render, signal } from 'pota'

function App() {
	const box = ref()
	const phase = signal('in')

	async function flip() {
		await animateClassTo(box(), 'in', 'out')
		phase.write('out')
		await animateClassTo(box(), 'out', 'in')
		phase.write('in')
	}

	return (
		<div>
			<button on:click={flip}>flip</button>
			<div use:ref={box} class="in">
				phase: {phase.read}
			</div>
		</div>
	)
}

render(<App />)
```
