---
title: Paginate a list
description: Slice items into pages with `next`/`previous` and totals.
category: use
api: paginate
import: pota/use/paginate
tags: [paginate, list]
---

`paginate(fetch, options)` returns a small object: `items` (a derived
of the current page's array), `currentPage`, `totalPages`, `next`,
`previous`, and predicate `hasNext` / `hasPrevious`. `fetch(start,
end)` may return synchronously or as a `Promise`; the items derived
unwraps either. `paginateValues(values, perPage)` is the in-memory
shorthand.

```jsx
import { paginateValues } from 'pota/use/paginate'
import { For } from 'pota/components'
import { render, signal } from 'pota'

const all = Array.from({ length: 23 }, (_, i) => `item ${i + 1}`)

function App() {
	const perPage = signal(5)
	const page = paginateValues(() => all, perPage.read)

	return (
		<div>
			<p>
				page {page.currentPage} / {page.totalPages}
			</p>
			<ul>
				<For each={page.items}>{x => <li>{x}</li>}</For>
			</ul>
			<button
				on:click={page.previous}
				prop:disabled={() => !page.hasPrevious()}
			>
				← prev
			</button>
			<button
				on:click={page.next}
				prop:disabled={() => !page.hasNext()}
			>
				next →
			</button>
		</div>
	)
}

render(<App />)
```
