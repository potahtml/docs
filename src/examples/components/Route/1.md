---
title: Client-side routing with Route
description: Render content when the URL matches a path.
category: components
api: Route
import: pota/components
tags: [Route, routing]
---

`<Route path="...">` renders its children when `location.pathname`
matches `path` (relative to any parent route). Routes nest by
declaration: a `<Route path="/users">` containing `<Route
path=":id">` matches `/users/42`. `path` accepts `:params` which
become available via the route context.

```jsx
import { render } from 'pota'
import { A, Route } from 'pota/components'

function App() {
	return (
		<div>
			<nav>
				<A href="/">home</A> · <A href="/about">about</A> · <A href="/users/42">user 42</A>
			</nav>

			<Route path="/">
				<h2>welcome</h2>
			</Route>
			<Route path="/about">
				<h2>about us</h2>
			</Route>
			<Route path="/users/:id">
				<h2>user profile</h2>
				<p>(open the URL — :id is captured)</p>
			</Route>
		</div>
	)
}

render(<App />)
```
