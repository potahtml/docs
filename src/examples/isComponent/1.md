---
title: Distinguish components from plain functions
description: Branch on whether a value is a JSX-marked component.
category: components
api: isComponent
import: pota
tags: [isComponent, introspection]
---

`isComponent(v)` returns `true` for things tagged via `markComponent`
— the renderer uses the same check to know whether to call a value
as a component (untracked) or treat it as a reactive expression. The
helper is exposed for component libraries that need to branch on
"this prop is a renderable" vs "this prop is a getter".

```jsx
import { isComponent, markComponent, render } from 'pota'

const Greeting = markComponent(() => <p>hello!</p>)
const counter = () => Math.random()

console.log(isComponent(Greeting)) // true
console.log(isComponent(counter)) // false

function Wrapper(props) {
	return (
		<div>
			{isComponent(props.body) ? props.body() : <p>{props.body}</p>}
		</div>
	)
}

render(<Wrapper body={Greeting} />)
```
