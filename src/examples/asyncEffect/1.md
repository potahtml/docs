---
title: Sequential async effect
description: Run an async side-effect and await the previous run before starting the next.
category: reactivity
api: asyncEffect
import: pota
tags: [asyncEffect, async]
---

`asyncEffect(fn)` calls `fn(prev)` where `prev` is the promise from
the previous run (or `undefined` on the first). Awaiting `prev` lets
you guarantee one run finishes before the next begins — useful for
chained network requests where overlapping is wrong.

```jsx
import { asyncEffect, render, signal } from 'pota'

function App() {
	const id = signal(1)
	const data = signal(null)

	asyncEffect(async prev => {
		await prev
		const current = id.read()
		const res = await fetch(`/api/items/${current}`)
		data.write(await res.json())
	})

	return (
		<div>
			<button on:click={() => id.update(n => n + 1)}>next</button>
			<pre>{() => JSON.stringify(data.read(), null, 2)}</pre>
		</div>
	)
}

render(<App />)
```
