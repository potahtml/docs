---
title: Batch a sequence of store mutations
description: Apply many writes to a `mutable` and let observers see one consistent state.
category: reactivity
api: batch
import: pota
tags: [batch, store, mutable]
---

Mutations against a `mutable` proxy notify observers per write. Wrap
a multi-step transition in `batch` so the result reaches readers as
a single transaction — invariants between fields hold even when the
intermediate state would be inconsistent.

```jsx
import { batch, render } from 'pota'
import { mutable } from 'pota/store'

const account = mutable({ debit: 100, credit: 0 })

function transfer(amount) {
	batch(() => {
		account.debit -= amount
		account.credit += amount
	})
}

function App() {
	return (
		<div>
			<p>debit: {() => account.debit}</p>
			<p>credit: {() => account.credit}</p>
			<button on:click={() => transfer(25)}>move 25</button>
		</div>
	)
}

render(<App />)
```
