---
title: Group multiple writes into one update
description: Coalesce several signal writes so dependent effects run once.
category: reactivity
api: batch
import: pota
tags: [batch, scheduling]
---

`batch(fn)` defers downstream re-runs until `fn` returns. Without it,
two writes back-to-back would fire two effect runs; inside a batch,
the effect runs once with the final state. Most user code paths
(event handlers, cleanups) already batch — reach for `batch` when
you're synthesizing your own update boundary.

```jsx
import { batch, effect, render, signal } from 'pota'

function App() {
	const first = signal('Ada')
	const last = signal('Lovelace')

	effect(() => {
		console.log(`hello, ${first.read()} ${last.read()}`)
	})

	return (
		<button
			on:click={() =>
				batch(() => {
					first.write('Grace')
					last.write('Hopper')
				})
			}
		>
			rename (single effect run)
		</button>
	)
}

render(<App />)
```
