---
title: Encapsulated counter — expose only what callers need
description: Hide the writer behind a hook-style function that returns just a reader and actions.
category: reactivity
api: signal
import: pota
tags: [signal, encapsulation, hook]
---

A common pattern is to keep `write`/`update` private and expose only
a reader plus named actions. The signal stays the source of truth
internally; consumers can't bypass the action API to write whatever
they like.

```jsx
import { render, signal } from 'pota'

function useCounter(initial = 0) {
	const n = signal(initial)
	return {
		count: n.read,
		increment: () => n.update(v => v + 1),
		decrement: () => n.update(v => v - 1),
		reset: () => n.write(initial),
	}
}

function App() {
	const counter = useCounter(10)

	return (
		<div>
			<p>{counter.count}</p>
			<button on:click={counter.increment}>+</button>
			<button on:click={counter.decrement}>−</button>
			<button on:click={counter.reset}>reset</button>
		</div>
	)
}

render(<App />)
```
