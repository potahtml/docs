---
title: Run code after mount
description: Defer imperative work until after the DOM is in place.
category: scheduler
api: ready
import: pota
tags: [ready, scheduler, mount]
---

`ready(fn)` schedules `fn` to run once the renderer has finished
mounting (after `onProps`, `onRef`, and `onMount`). Use it for
imperative work that needs the node to be in the DOM — measuring
sizes, focusing inputs, attaching third-party widgets.

```jsx
import { ready, ref, render } from 'pota'

function App() {
	const input = ref()

	ready(() => {
		input().focus()
		input().select()
	})

	return (
		<input
			use:ref={input}
			value="select me on mount"
		/>
	)
}

render(<App />)
```
