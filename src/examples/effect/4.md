---
title: Debounced effect
description: Defer reactive side-effects until inputs stop changing for a moment.
category: reactivity
api: effect
import: pota
tags: [effect, debounce, useTimeout]
---

When an effect should react to rapid signal changes (typing in a
search box, dragging a slider) but the side-effect is expensive,
debounce it. `useTimeout` from `pota/use/time` is owner-aware:
its returned timer cancels automatically when the surrounding scope
disposes. Restarting the timer on every change implements debounce.

```jsx
import { effect, render, signal } from 'pota'
import { useTimeout } from 'pota/use/time'

function App() {
	const query = signal('')
	const result = signal('—')

	effect(() => {
		const q = query.read()
		const timer = useTimeout(() => {
			result.write(q ? `searched for "${q}"` : '—')
		}, 400)
		timer.start()
	})

	return (
		<div>
			<input
				placeholder="type a query…"
				on:input={e => query.write(e.currentTarget.value)}
			/>
			<p>{result.read}</p>
		</div>
	)
}

render(<App />)
```
