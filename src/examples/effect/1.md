---
title: Log a signal whenever it changes
description: Run a side-effect every time tracked state updates.
category: reactivity
api: effect
import: pota
tags: [effect, basics]
---

`effect(fn)` runs `fn` once and again every time anything it read
changes. It's the right tool for side-effects that mirror reactive
state into the outside world (logging, persistence, third-party
libraries) — but prefer `memo`/`derived` when you can derive a value
instead of imperatively pushing it.

```jsx
import { effect, render, signal } from 'pota'

function App() {
	const count = signal(0)

	effect(() => {
		console.log('count is now', count.read())
	})

	return (
		<button on:click={() => count.update(n => n + 1)}>
			{count.read}
		</button>
	)
}

render(<App />)
```
