---
title: Turn an object's keys into signals in place
description: Make existing properties reactive without proxying the whole object.
category: store
api: signalify
import: pota/store
tags: [signalify, signals]
---

`signalify(target, keys?)` rewrites properties on `target` so reads
and writes go through signals. It's not recursive — only the named
keys (or all own keys if omitted) become tracked. Useful for
attaching reactivity to plain objects (or class instances) without
the proxy overhead of `mutable`.

```jsx
import { signalify } from 'pota/store'
import { effect, render } from 'pota'

const config = signalify({ theme: 'light', density: 'comfortable' })

effect(() => {
	console.log('theme is now', config.theme)
})

function App() {
	return (
		<div>
			<button on:click={() => (config.theme = 'dark')}>
				dark
			</button>
			<button on:click={() => (config.theme = 'light')}>
				light
			</button>
			<p>theme: {() => config.theme}</p>
		</div>
	)
}

render(<App />)
```
