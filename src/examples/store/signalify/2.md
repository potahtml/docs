---
title: Signalify only specific keys
description: Make `count` reactive while leaving other fields plain.
category: store

---

`signalify(target, keys)` accepts an array of keys to convert.
Useful when most of an object's fields are static (or change in
batches you'd rather control imperatively) but a handful need
fine-grained reactivity. Keys not yet present can also be listed —
the property gets created with signal-backed accessors immediately.

```jsx
import { signalify } from 'pota/store'
import { effect, render } from 'pota'

const cart = signalify(
	{
		userId: 'u-123', // stays plain
		count: 0, // becomes signal
		total: 0, // becomes signal (creates the key now)
	},
	['count', 'total'],
)

effect(() => {
	console.log(`${cart.count} item(s), total ${cart.total}`)
})

function App() {
	return (
		<div>
			<p>user: {cart.userId} (not reactive)</p>
			<p>
				count: {() => cart.count} — total: {() => cart.total}
			</p>
			<button
				on:click={() => {
					cart.count++
					cart.total += 9.99
				}}
			>
				add item
			</button>
		</div>
	)
}

render(<App />)
```
