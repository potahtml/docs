---
title: Replace a store with new data
description: Swap to a fresh shape, removing keys not present in the source.
category: store
api: replace
import: pota/store
tags: [replace, reconcile]
---

`replace(target, source, keys?)` is `merge` plus deletion: keys in
`target` that aren't in `source` are removed. With a `key` matcher,
list reordering is performed in place — items with the same `id`
keep their reference but move to the new position.

```js
import { mutable, replace } from 'pota/store'

const state = mutable({
	a: true,
	q: [{ id: 1, name: 'Quack' }, { id: 2, name: 'Duck' }],
})

const ref = state.q[0]

replace(
	state,
	{ q: [{ id: 1, lastName: 'Murci' }] },
	{ q: { key: 'id' } },
)

console.log(state) // → { q: [{ id: 1, name: 'Quack', lastName: 'Murci' }] }
console.log(state.q[0] === ref) // true — id:1 kept its reference
```
