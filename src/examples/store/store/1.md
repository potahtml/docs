---
title: Store with a batched setter
description: Use the `[store, setStore]` pair for typed reads and grouped writes.
category: store
api: store
import: pota/store
tags: [store, batch, setter]
---

`store(source, clone?)` returns `[store, setStore]`: `store` is the
mutable proxy typed as `DeepReadonly` (so the only sanctioned
mutation path is the setter), and `setStore(fn)` runs `fn(draft)`
inside `batch()` so multiple writes flush as one. Direct writes to
`store` would still succeed at runtime — TypeScript is what enforces
the contract.

```jsx
import { store } from 'pota/store'
import { effect, render } from 'pota'

const [user, setUser] = store({ name: 'ada', age: 0, role: 'engineer' })

effect(() => {
	console.log('user is now', user.name, user.age, user.role)
})

function App() {
	return (
		<button
			on:click={() =>
				setUser(u => {
					u.name = 'grace'
					u.age = 30
					u.role = 'rear admiral'
				})
			}
		>
			rename and promote (one effect run)
		</button>
	)
}

render(<App />)
```
