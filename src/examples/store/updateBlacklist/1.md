---
title: Extend the mutation blacklist for another window
description: Teach `mutable` about constructors from an iframe / shadow realm.
category: store
api: updateBlacklist
import: pota/store
tags: [updateBlacklist, blacklist, iframe]
---

`mutable` refuses to wrap built-ins like `Date`, `RegExp`, or DOM
nodes — they live on a constructor blacklist sourced from
`globalThis`. When you load code from another realm (an iframe, a
worker context's `globalThis`), call `updateBlacklist(otherWindow)`
once so its constructors are added — otherwise instances from that
realm could slip through and get proxied.

```js
import { updateBlacklist } from 'pota/store'

const frame = document.createElement('iframe')
document.body.append(frame)

updateBlacklist(frame.contentWindow)

// now `mutable(...)` will treat instances of frame.contentWindow.Date
// as opaque, just like instances of the host's Date.
```
