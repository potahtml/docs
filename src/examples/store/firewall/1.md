---
title: Run a function in a firewalled scope
description: Wrap effectful code so its writes don't propagate to outer trackers.
category: store
api: firewall
import: pota/store
tags: [firewall, scope]
---

`firewall(fn)` runs `fn` inside a memo whose result chains through
`usePrevious` — used to break a reactive feedback loop where
downstream code would otherwise re-trigger upstream tracking. Reach
for it when you need to compute a value reactively but want to
isolate side-effects to that one pass.

```js
import { firewall, mutable } from 'pota/store'
import { effect } from 'pota'

const counter = mutable({ value: 0 })

effect(() => {
	firewall(() => {
		// reads inside the firewall don't re-subscribe the outer effect
		const v = counter.value
		console.log('snapshot inside firewall:', v)
	})
})

// changing this still re-runs the outer effect (the read happens inside)
counter.value++
```
