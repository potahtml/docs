---
title: Reset specific keys to defaults
description: Overwrite only the keys defined in source, leave the rest alone.
category: store
api: reset
import: pota/store
tags: [reset, defaults]
---

`reset(target, source)` walks `source` and writes each value into
`target` — but unlike `replace` it doesn't delete keys absent from
source. Useful for "restore defaults" affordances, or for clearing a
slice of state without touching the rest.

```js
import { mutable, reset } from 'pota/store'

const state = mutable({
	user: { name: 'Ada' },
	form: { name: 'edited', age: 99 },
})

const defaults = { form: { name: '', age: 0 } }

reset(state, defaults)

console.log(state) // → { user: { name: 'Ada' }, form: { name: '', age: 0 } }
```
