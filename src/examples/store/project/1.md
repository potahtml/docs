---
title: Copy-on-write projection of a store
description: Edit a draft view without mutating the underlying source.
category: store
api: project
import: pota/store
tags: [project, projection, draft]
---

`project(value)` returns a mutable proxy that *reads through* to
`value` but writes only to its own copy. Each `project()` call gets
its own proxy store, so two projections of the same source stay
independent — useful for editable drafts, undo stacks, "what-if"
scenarios.

```jsx
import { mutable, project } from 'pota/store'
import { render } from 'pota'

const original = mutable({ name: 'Ada', age: 30 })
const draft = project(original)

function App() {
	return (
		<div>
			<p>
				original: {() => original.name}, {() => original.age}
			</p>
			<p>
				draft: {() => draft.name}, {() => draft.age}
			</p>
			<button on:click={() => (draft.age += 1)}>
				bump draft age (original unchanged)
			</button>
			<button on:click={() => (original.name = 'Grace')}>
				rename original (draft sees it through)
			</button>
		</div>
	)
}

render(<App />)
```
