---
title: Merge new data into a store
description: Layer fresh values onto a target without losing existing object references.
category: store
api: merge
import: pota/store
tags: [merge, reconcile]
---

`merge(target, source, keys?)` walks `source` and writes into
`target`, preserving existing object references when the structure
matches. The optional `keys` argument tells `merge` how to identify
list items so that updating an item-by-id doesn't replace the whole
array element.

```js
import { merge, mutable } from 'pota/store'

const state = mutable({
	items: [
		{ id: 1, name: 'Quack' },
		{ id: 2, name: 'Duck' },
	],
})

const ref = state.items[0]

// re-fetched data with overlapping ids
merge(
	state,
	{
		items: [
			{ id: 1, score: 10 },
			{ id: 3, name: 'Goose' },
		],
	},
	{ items: { key: 'id' } },
)

console.log(state.items[0] === ref) // true — same reference
console.log(state.items) // → [{id:1, name:'Quack', score:10}, {id:2, name:'Duck'}, {id:3, name:'Goose'}]
```
