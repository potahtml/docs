---
title: Reactive Map and Set
description: `mutable` works on Maps and Sets, not just objects and arrays.
category: store
api: mutable
import: pota/store
tags: [mutable, Map, Set]
---

`mutable` proxies `Map` and `Set` instances too: `set`/`delete`/
`clear`/`add` notify observers, and reads (`get`/`has`/`size`,
iteration) subscribe. Useful when you want O(1) lookup or
guaranteed uniqueness with full reactivity.

```jsx
import { mutable } from 'pota/store'
import { For } from 'pota/components'
import { render, signal } from 'pota'

const tags = mutable(new Set(['ui', 'docs']))
const counts = mutable(new Map([
	['ui', 12],
	['docs', 4],
]))

function App() {
	const draft = signal('')

	function addTag() {
		const t = draft.read().trim()
		if (!t) return
		tags.add(t)
		counts.set(t, (counts.get(t) ?? 0) + 1)
		draft.write('')
	}

	return (
		<div>
			<input
				prop:value={draft.read}
				on:input={e => draft.write(e.currentTarget.value)}
			/>
			<button on:click={addTag}>add</button>

			<p>tags ({() => tags.size}):</p>
			<ul>
				<For each={() => [...tags]}>
					{tag => (
						<li>
							{tag} ({() => counts.get(tag) ?? 0})
						</li>
					)}
				</For>
			</ul>
		</div>
	)
}

render(<App />)
```
