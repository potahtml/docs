---
title: Cloned mutable to isolate from source
description: Use `clone: true` so mutations don't affect the original object.
category: store
api: mutable
import: pota/store
tags: [mutable, clone, isolation]
---

By default `mutable(source)` shares structure with `source` —
external code that holds the original reference can mutate around
the proxy's tracking. Pass `true` as the second argument to deep-copy
first; the proxy and the original become independent.

```js
import { mutable } from 'pota/store'

const original = { count: 0, nested: { value: 1 } }
const m = mutable(original, true)

m.count = 42
m.nested.value = 99

console.log(original.count) // 0 (unchanged)
console.log(original.nested.value) // 1 (unchanged)
console.log(m.count) // 42
console.log(m.nested.value) // 99
```
