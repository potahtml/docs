---
title: Synchronous prop plugin (no microtask defer)
description: Pass `onMicrotask: false` so the plugin runs at element creation time.
category: props
api: propsPlugin
import: pota
tags: [propsPlugin, sync, onMicrotask]
---

By default, prop plugins are queued on the `onProps` microtask so
children and refs are in place before they fire. Pass `false` as the
third argument to run the handler synchronously when the prop is
first applied — useful when the directive needs to set the
attribute *before* the element is connected.

```jsx
import { propsPlugin, render, signal } from 'pota'

propsPlugin(
	'use:initial-color',
	(node, value) => {
		node.style.color = value
	},
	false,
)

function App() {
	const color = signal('rebeccapurple')
	return (
		<div>
			<p use:initial-color={color.read()}>colored at creation</p>
			<button on:click={() => color.write('crimson')}>
				doesn't update — plugin only ran once
			</button>
		</div>
	)
}

render(<App />)
```
