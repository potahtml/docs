---
title: Patch a list while preserving identity
description: Reuse existing items by `id` when fresh data arrives from a server.
category: reactivity
api: externalSignal
import: pota
tags: [externalSignal, list, id]
---

`externalSignal(initial)` behaves like `signal` for reads, but its
`write(fresh)` patches the array by `id`: items already present and
deep-equal stay as the same reference, only changed/new items become
new references. Cheap way to merge polled or refreshed data without
reaching for a full `store`.

```jsx
import { externalSignal, render } from 'pota'
import { For } from 'pota/components'

function App() {
	const todos = externalSignal([
		{ id: 1, text: 'buy milk', done: false },
		{ id: 2, text: 'walk dog', done: true },
	])

	function refresh() {
		todos.write([
			{ id: 1, text: 'buy milk', done: true },
			{ id: 2, text: 'walk dog', done: true },
			{ id: 3, text: 'water plants', done: false },
		])
	}

	return (
		<div>
			<button on:click={refresh}>refresh</button>
			<ul>
				<For each={todos.read}>
					{todo => (
						<li>
							{todo.text} — {todo.done ? '✓' : '…'}
						</li>
					)}
				</For>
			</ul>
		</div>
	)
}

render(<App />)
```
