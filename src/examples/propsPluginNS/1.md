---
title: Register a namespaced prop family
description: Define a prefix (e.g. `track:`) so any sub-prop under it routes to your plugin.
category: props
api: propsPluginNS
import: pota
tags: [propsPluginNS, plugin, namespace]
---

`propsPluginNS(ns, fn)` registers a plugin that fires for *any* prop
under that namespace. The handler receives `(node, localName, value,
ns)`, so `track:click` and `track:hover` route to the same plugin
with `localName === 'click'` / `'hover'`. Built-in `on:`, `prop:`,
`class:`, `style:`, and `use:` work the same way.

```jsx
import { propsPluginNS, render } from 'pota'

propsPluginNS('track', (node, localName, value) => {
	node.addEventListener(localName, () => {
		console.log('analytics:', value, 'on', node.tagName)
	})
})

function App() {
	return (
		<div>
			<button track:click="cta-pressed">primary</button>
			<a track:click="link-clicked" href="#">
				link
			</a>
		</div>
	)
}

render(<App />)
```
