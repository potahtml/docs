title: Compiler-less templates with the xml tag
description: Render UI from tagged template literals — no build step required.

`xml\`…\`` is a tagged template that parses HTML-like markup at
runtime and produces a JSX-equivalent element. Templates are parsed
as `text/xml`, so void elements need a trailing slash (`<br/>`),
every open tag must close, and attribute values must be quoted.
Ill-formed input renders a visible `parsererror` (it does not
throw).

```js
import { signal } from 'pota'
import { render } from 'pota'
import { xml } from 'pota/xml'

const count = signal(0)

const App = xml`
	<div>
		<p>count: ${count.read}</p>
		<button on:click=${() => count.update(n => n + 1)}>+</button>
		<button on:click=${() => count.write(0)}>reset</button>
	</div>
`

render(App)
```
