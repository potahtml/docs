---
title: Catch errors from descendants
description: Route synchronous and reactive errors to a handler.
category: reactivity
api: catchError
import: pota
tags: [catchError, error, boundary]
---

`catchError(fn, handler)` runs `fn` in a scope where any error from
its descendants — synchronous throws, effect failures, rejected
async work in `action`/`derived` — is routed to `handler` instead of
the console. It's the building block behind the `Errored` component;
reach for it when you need a programmatic boundary outside JSX.

```jsx
import { catchError, render, root, signal } from 'pota'

function App() {
	const message = signal('not yet')

	root(() => {
		catchError(
			() => {
				throw new Error('boom')
			},
			err => message.write(`caught: ${err.message}`),
		)
	})

	return <p>{message.read}</p>
}

render(<App />)
```
