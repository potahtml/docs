---
title: Wait for a derived chain to commit
description: Branch UI on whether an async derived has resolved at least once.
category: reactivity
api: isResolved
import: pota
tags: [isResolved, derived, async]
---

`isResolved(d, ...)` returns `true` once every passed derived has
committed at least once. Async chains (containing promises) start
unresolved — the returned value is whatever the chain's last
committed run produced, which is `undefined` initially. `isResolved`
is the way to render a loading state without throwing on the first
read.

```jsx
import { derived, isResolved, render, signal } from 'pota'

function App() {
	const id = signal(1)

	const user = derived(
		() => `https://jsonplaceholder.typicode.com/users/${id.read()}`,
		url => fetch(url),
		res => res.json(),
	)

	return (
		<div>
			<button on:click={() => id.update(n => n + 1)}>next</button>
			<p>
				{() =>
					isResolved(user)
						? user().name
						: 'loading…'
				}
			</p>
		</div>
	)
}

render(<App />)
```
